## v1.2.0 - 2024-10-15

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "bugfix"]}>

## New Features

- Added real-time collaboration for shared documentation spaces
- Introduced custom brand colors (default: `#3B82F6`) for personalized themes
- Enabled export to PDF and Markdown formats directly from the editor

## Improvements

- Enhanced search with fuzzy matching and syntax highlighting
- Optimized loading times by `>40%` for large documentation sites

## Bug Fixes

- Fixed rendering issues with nested code blocks
- Resolved authentication token refresh failures during long sessions

</Update>

## v1.1.0 - 2024-09-20

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "improvement"]}>

## New Features

- Integrated version control with GitHub sync
- Added component preview mode for MDX elements like `<Callout>` and `<Steps>`

## Improvements

- Streamlined sidebar navigation with drag-and-drop reordering
- Better mobile responsiveness for documentation previews

## Deprecations

- Legacy `{api_key}` format deprecated; migrate to `API_KEY`

</Update>

## v1.0.0 - 2024-08-10

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## New Features

- Core MDX documentation engine with 15+ components
- Full-text search across all pages
- YAML frontmatter validation on save

## Breaking Changes

- Updated default theme; custom CSS now required for overrides

## Initial Setup

- Basic authentication via `https://auth.example.com`
- API base: `https://api.example.com/v1`

</Update>

## Migration Steps

Follow these steps to update your project after each release.

<Steps>
  <Step title="Check Compatibility" icon="check-circle">

    Review the tags in each `<Update>`:

    - `breaking`: Test thoroughly
    - `feature`: Enable new flags
    - `bugfix`: Deploy immediately

  </Step>
  <Step title="Update Package" icon="download">

    <CodeGroup tabs="npm,yarn">
    ````bash
    npm install kristina-goshko-docs@latest
    ````
    ````bash
    yarn add kristina-goshko-docs@latest
    ````
    </CodeGroup>

  </Step>
  <Step title="Run Migrations" icon="settings">

    Execute any required scripts:

    ````bash
    npx goshko-migrate v1.2.0
    ````

  </Step>
  <Step title="Test Deployment" icon="rocket">

    Preview at `https://dashboard.example.com` and verify components render correctly.

  </Step>
</Steps>
